# Claw Design

一句话把内容变成可交付的设计产物。

🌐 [官网](https://agentos.site/claw-design.html)

Claw Design 是 OpenClaw 里的 AI 设计引擎。你在 OpenClaw 里说一句需求，它会判断你要做什么，补齐必要信息，生成可展示、可编辑、可归档、可复用的结果。

---

## 你大概率正在被这些问题卡住

周五下午三点，老板说“下周一给客户做个方案演示”。内容你有，但拆成演示文稿要半天，调版式又半天，导出 PDF 发现格式乱了再修半天。周末没了。

产品经理画了个交互原型想验证思路，找设计排期要两周。等排上了，需求已经变了三轮。

运营同事每月做数据报告，数据早就跑出来了，但做成图表、仪表盘、信息图，每次都是从零开始拖拽。

技术负责人要画架构图给新人看，Visio 太重，白板太随意，AI 工具生成的东西一股“紫色渐变 + emoji 图标”的塑料味，拿不出手。

共同的困境：**内容早就有了，卡在“变成可交付产物”这一步。**

## 核心能力

**一句话触发完整链路**
意图识别 → 必要时追问 → 上下文收束 → 设计生成 → 质量门禁 → 导出交付。你只管说要什么，链路自动跑完。

**不确定时会追问你**
需求太模糊时，Claw Design 会先问清楚，不会硬猜。比如你只说“做个发布会页面”，它可能会追问受众、主视觉、转化目标。最多追问 2 轮；能继续推进的缺口，会用默认骨架先跑。

**生成完还能继续改**
不满意不用重来。直接说“把标题换个颜色”“第二页改得更像高管汇报”“图表换成桑基图”，Claw Design 会基于上一次结果做增量修订，保留已有结构，只改你点名的地方。

**品牌风格能管住**
Theme Pack / Brand Kit 会把颜色、字体、标识、间距、版式规则收成一套可复用的品牌上下文。企业团队做客户方案、月报、落地页时，不用每次重新解释品牌规范，也不用担心一会儿蓝、一会儿紫、一会儿毛玻璃。

**12 类设计产物，覆盖日常交付场景**

| 产物类型 | 差异化价值 |
|---------|-----------|
| 演示文稿 | 文本直接变 PPT，支持翻页演示和可编辑导出，不用再手工拆页 |
| 图表与数据可视化 | 柱状图、桑基图、热力图等一句话生成，数据有了图就有了 |
| 架构图与系统图 | 节点、分组、连接关系清晰可读，深色浅色主题切换，矢量可复用 |
| 流程图与时序图 | 泳道、决策分支、参与者顺序自动编排，复杂流程按模块拆段不拥挤 |
| 海报与宣传图 | 纯代码驱动构图，不依赖外部图片生成服务，文字层级和装饰图形自动编排 |
| 落地页 | 首屏、卖点、分区、CTA 一步到位，直接分享或继续改版 |
| 界面稿与线框图 | 桌面端和移动端画布，低保真到高保真两档，快速验证产品想法 |
| 仪表盘 | 指标卡、趋势图、明细区组合，同一份数据可出演示型和管理型两种版式 |
| 信息图 | 复杂内容压缩成单页，层级、步骤、对比、数字亮点一目了然 |
| 逻辑关系图 | 长文本和论证结构拆解为递进、并列、对比、循环等可视化表达 |
| 交互原型 | 可点击的 HTML 原型，支持页面跳转和状态切换，浏览器打开就能演示 |
| 多产物联动 | 一次需求同时生成 PPT、海报、落地页、信息图等配套物料 |

**渲染层基于 HTML/CSS/SVG/JS**
核心链路不依赖外部图片生成服务。交付物是标准 Web 技术栈，OpenClaw 可以继续预览、导出、归档和分发。

**多格式导出**
主交付是 HTML + PPTX，同时支持 PDF、PNG、SVG。同一份内容，演示用 HTML，归档用 PDF，编辑用 PPTX，传播用 PNG。

## Slop 黑名单：干掉 AI 味

这是 Claw Design 的差异化杀手锏。

市面上 AI 设计工具的通病：生成的东西一眼就能看出是 AI 做的。紫蓝渐变背景、emoji 当图标、所有按钮都是药丸形、千篇一律的三列推荐语卡片、到处都是毛玻璃效果……

Claw Design 内置 20+ 条 Slop 检测规则，在交付前自动扫描产物，命中即阻断重做：

- `generic-purple-gradient` — 紫蓝渐变大面积背景
- `emoji-as-icon` — 用 emoji 代替专业图标
- `glassmorphism-overuse` — 毛玻璃效果滥用
- `all-pill-buttons` — 所有按钮都是药丸形
- `uniform-card-grid` — 千篇一律的等尺寸卡片网格
- `generic-hero-layout` — “大标题 + 副标题 + CTA 居中”的万能首屏
- `dark-neon-combo` — 深色背景配霓虹色的赛博朋克审美
- `centered-everything` — 全页面居中对齐，毫无层级变化
- ……以及更多

severity=block 的规则命中后，系统自动重新生成（最多 3 次），并在 prompt 中注入规避指导。你也可以在 OpenClaw 里查看、添加、禁用规则，或者调整严重级别。

结果：交付物看起来像人做的，拿得出手。

## 在 OpenClaw 里使用

### 前提

- 已安装并启用 OpenClaw。
- OpenClaw 已配置可用的文本 LLM。Claw Design 的意图识别、澄清问题、内容编排和设计生成都要调用文本 LLM。
- Node.js 18+。这是 OpenClaw 运行 Claw Design 包时需要的基础环境。
- 图像生成、视频生成、视觉理解都不用额外配置。没配这些能力，核心设计链路照样能跑。

### 直接说需求

在 OpenClaw 里发消息：

```text
帮我做一个关于 AI 趋势的演示文稿，适合给客户讲，风格稳一点。
```

也可以更具体：

```text
把这份 Q2 经营数据做成管理层仪表盘，突出营收、成本、续费率和风险客户。
```

```text
做一个微服务系统架构图，给新同事 onboarding 用，节点分层清楚一点。
```

```text
用我们公司的品牌色做一个发布会落地页，首屏要能直接讲清新品价值。
```

### 生成后继续改

产物出来后，继续用自然语言改：

```text
把标题换成深蓝色，第一页少一点装饰。
```

```text
第二页图表太挤了，拆成两页。
```

```text
整体更像企业客户提案，不要太互联网。
```

Claw Design 会按你的新指令修改已有结果，不会把整套东西推倒重来。

## 命令行使用（CLI）

装完包后会得到一个 `claw-design` 命令。`generate` 需要文本 LLM：可以由已配置文本 LLM 的 OpenClaw 注入，也可以在程序内通过 `createPipeline(theme, { classifierProvider })` 手动注入。没有 LLM 时，CLI 不会猜测意图，会给出引导性报错；`slop-list`、`--help`、`--version` 仍可直接使用。

### 安装

```bash
npm install -g @self-evolving-harness/claw-design
```

也可以装到项目里，用 `npx claw-design` 调用。

### 命令一览

```bash
claw-design generate "<一句话需求>"   # 生成设计制品（HTML/PPTX 到输出目录）
claw-design "<一句话需求>" -o ./out    # generate 可省略，-o 指定输出目录
claw-design plugin list               # 列出已安装插件
claw-design plugin install ./plugins/x # 安装本地插件
claw-design slop-list                  # 查看 Slop 黑名单规则状态
claw-design --help                     # 查看帮助
claw-design --version                  # 查看版本
```

### 生成示例

```bash
claw-design generate "帮我做一个关于 AI 趋势的演示文稿" -o ./output
```

生成成功后，终端会打印产物路径：主交付 HTML、可编辑 PPTX，以及质量门禁结论。
用浏览器打开 HTML 即可预览，PPTX 可在 PowerPoint/Keynote 里继续编辑。

```bash
open ./output/index.html      # macOS
xdg-open ./output/index.html  # Linux
```

### 关于意图识别（重要）

`generate` 的意图识别走文本 LLM 语义理解，不做关键词匹配降级。CLI 本身不内置 LLM：

- 在已配置文本 LLM 的 **OpenClaw** 里使用时，宿主会注入 LLM，`generate` 正常工作。
- 直接 `npm` 装的裸 CLI 没有注入 LLM 时，`generate` 不会瞎猜意图，而是报错并提示你去 OpenClaw 使用，或在程序内通过 `createPipeline(theme, { classifierProvider })` 注入文本 LLM classifier。
- `slop-list`、`--help`、`--version` 不依赖 LLM，任何环境都能直接用。

## 使用场景

**方案汇报 / 客户演示**
内容写好了，周五下午一句话变成演示文稿。HTML 在线演示，PPTX 留给客户编辑，PDF 归档备查。

**数据报告 / 经营分析**
每月的数据跑出来后，直接生成图表、仪表盘或信息图。不用再开 Excel 拖图表、截图、贴 PPT。

**技术文档可视化**
系统说明变架构图，业务规则变流程图，API 调用链变时序图。给新人看、给客户讲、写文档用。

**产品构想验证**
想法阶段就能出可点击的交互原型，不用等设计排期。验证完再决定要不要正式投入。

**活动运营 / 品牌传播**
活动信息变海报，产品介绍变落地页，品牌内容变宣传图。急活当天就能交。

**团队缺设计资源**
没有专职设计师的小团队，日常交付靠这个兜底。质量门禁保证产出不翻车。

## 故障排查

**OpenClaw 没反应**
先确认 OpenClaw 正常运行，再确认当前会话能调用文本 LLM。

**生成结果跑偏**
继续说清楚要改哪里。比如“保留结构，只把风格改成企业蓝”“第三页改成流程图”。Claw Design 会走增量修订。

**风格不统一**
补充品牌规则：主色、辅助色、字体、Logo 使用规则、禁用风格。后续任务会把这些规则收进 Theme Pack / Brand Kit。

**导出格式不对**
说明你要的用途：演示、编辑、归档、传播。Claw Design 会优先给对应格式。
