## FR-A09：主动交互式需求澄清

### TC-A09-01
- 关联 AC：AC1
- 前置条件：用户发送模糊设计意图（如"帮我做个PPT"，未指定受众、风格、内容范围）
- 测试步骤：
  1. 向系统输入模糊设计请求
  2. 观察 routing 阶段系统输出
- 预期结果：系统生成结构化提问清单，覆盖目标受众、风格偏好、品牌调性、内容范围、交付格式、色彩倾向、信息密度、使用场景中至少 3 项关键决策因素

### TC-A09-02
- 关联 AC：AC2
- 前置条件：用户意图已明确，进入 execution 阶段，存在版式选择、信息层级等关键设计决策未确认
- 测试步骤：
  1. 提供明确意图进入 execution 阶段
  2. 观察方案生成前系统行为
- 预期结果：系统对关键设计决策发起确认式提问，用户确认/修正后才继续生成，未确认前不输出设计方案

### TC-A09-03
- 关联 AC：AC3
- 前置条件：设计方案已生成，进入 quality 阶段，存在品牌合规不确定或内容取舍歧义
- 测试步骤：
  1. 生成含存疑项的设计方案
  2. 观察交付打包前系统行为
- 预期结果：系统对存疑项发起最终确认提问，用户回应后才进入交付打包，未回应前不输出最终交付物

### TC-A09-04
- 关联 AC：AC4
- 前置条件：系统触发任一阶段的澄清提问
- 测试步骤：
  1. 触发澄清流程
  2. 检查提问输出的数据结构
- 预期结果：提问以工具化接口输出，每条包含：问题文本、决策因素类型、可选默认值、跳过条件四个字段

### TC-A09-05
- 关联 AC：AC5
- 前置条件：澄清流程进行中，部分 MUST 级缺口未回填
- 测试步骤：
  1. 触发澄清，保留 MUST 级缺口未回答
  2. 尝试推进到下一阶段
- 预期结果：系统阻止进入下一阶段；当所有 MUST 级缺口已回填（或用户确认默认值/发出"直接生成"指令）后，系统放行

### TC-A09-06
- 关联 AC：AC6
- 前置条件：单阶段内已完成 2 轮提问，仍有未回填缺口
- 测试步骤：
  1. 在同一阶段连续触发 2 轮提问
  2. 第 2 轮后仍保留未回填缺口
  3. 观察系统行为
- 预期结果：系统不发起第 3 轮提问，对剩余未回填缺口采用默认值继续推进

### TC-A09-07
- 关联 AC：AC7
- 前置条件：检查 skill 层实现方式
- 测试步骤：
  1. 查看澄清规则的注入方式
  2. 确认是否通过 prompt 注入植入各阶段执行上下文
- 预期结果：澄清模板和决策因素词表由 skill 层提供，通过 prompt 注入方式植入各阶段执行上下文，路由逻辑中无硬编码澄清规则

## FR-B13：交互原型生成

### TC-B13-01
- 关联 AC：AC1
- 前置条件：用户请求生成可交互的产品原型
- 测试步骤：
  1. 输入"生成一个可点击的登录注册原型"
  2. 检查生成的 HTML 文件
- 预期结果：HTML 包含可点击 UI 组件（按钮、表单、导航栏等），组件带 JS 事件绑定，点击按钮有反馈，表单可交互，状态可切换

### TC-B13-02
- 关联 AC：AC2
- 前置条件：用户请求包含多个页面的原型
- 测试步骤：
  1. 请求生成含首页、列表页、详情页的原型
  2. 在浏览器中打开生成的 HTML
  3. 点击导航链接或按钮
- 预期结果：单文件内通过 hash router 实现多页面，点击链接/按钮可在页面间跳转，URL hash 变化对应页面切换

### TC-B13-03
- 关联 AC：AC3
- 前置条件：原型包含需要状态切换的组件（tab、折叠面板、弹窗）
- 测试步骤：
  1. 生成含 tab 切换和弹窗的原型
  2. 点击 tab 标签
  3. 点击触发弹窗的按钮
- 预期结果：组件基于 data-state 属性管理状态，tab 点击后切换显示内容，弹窗按钮点击后弹窗显示，再次操作可关闭

### TC-B13-04
- 关联 AC：AC4
- 前置条件：检查原型生成的技术实现方式
- 测试步骤：
  1. 触发原型生成
  2. 检查 prototype skill 提供的模板和 LLM 填充的内容
- 预期结果：prototype skill 提供交互骨架模板（路由模板、组件模板、状态管理模板），LLM 负责填充具体内容、布局和业务逻辑，两者职责分离

### TC-B13-05
- 关联 AC：AC5
- 前置条件：原型已生成
- 测试步骤：
  1. 将生成的 HTML 文件直接在浏览器中打开（file:// 协议）
  2. 测试所有交互功能
- 预期结果：原型为独立 HTML 文件，无需构建工具、无需 npm install、无需本地服务器，直接双击打开即可完整交互

### TC-B13-06
- 关联 AC：AC6
- 前置条件：原型已生成
- 测试步骤：
  1. 在桌面端浏览器（1920px 宽）打开原型
  2. 在移动端视口（375px 宽）打开同一原型
- 预期结果：原型在两种视口下均可正常显示和交互，通过响应式布局适配或提供独立画布切换入口
