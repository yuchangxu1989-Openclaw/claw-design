# ADR-004: 零模态依赖视觉实现

**日期**：2026-04-19
**状态**：accepted
**决策者**：OpenClaw（sa-01 子Agent）

## 上下文（Context）

Claw Design 的核心约束：文本 LLM 是唯一必要的 AI 依赖，图像生成、视频生成、视频理解等模态模型均为可选增强项（NFR-12）。同时需要支撑海报、架构图、信息图等视觉密集型产物，视觉表现力不能因为零模态依赖约束而大幅退化。所有 Skill 的渲染层基于 HTML/CSS/SVG/JS，不调用任何模态工具，用户只需配置文本 LLM 即可使用全部设计能力。

## 备选方案（Options Considered）

### 方案 A：HTML/CSS/SVG/JS 渲染 + Mermaid 图表

- 描述：用 CSS 实现背景、渐变、阴影、动画等视觉效果，SVG 实现图标、图形、装饰元素，Mermaid 实现流程图/时序图/类图等结构化图表，JS 处理交互和动态效果
- 优点：全部本地渲染，零模态依赖，结果确定性高，矢量输出可缩放，用户零额外配置成本
- 缺点：无法生成照片级图像，视觉丰富度受限于代码渲染表达能力
- 成本/复杂度：需要积累 CSS/SVG 模式库，但无运行时成本

### 方案 B：预置图片资产库

- 描述：内置一批通用图片素材（图标、背景、插图），按需引用
- 优点：视觉丰富度高于纯代码渲染
- 缺点：分发包体积膨胀（违反 ≤ 50MB 约束），素材覆盖面有限，版权风险
- 成本/复杂度：素材采购/制作成本高，维护负担重

### 方案 C：可选外部模态 API

- 描述：默认不调用，但允许用户配置图像生成/视频生成等模态服务
- 优点：视觉上限最高
- 缺点：引入外部依赖和成本，结果不确定性增加，与产品定位（零模态配置成本）冲突
- 成本/复杂度：需要维护 API 适配层

## 决策（Decision）

选择方案 A 为主渲染路径，方案 C 作为可选增强。

主渲染路径（HTML/CSS/SVG/JS + Mermaid）独立完成全部产出，无功能缺失。模态模型（图像生成等）作为可选 Provider 接入，配置后自动增强视觉表现力上限，未配置时主渲染路径不受影响。

具体技术手段：
- CSS：渐变背景、box-shadow、border-radius、clip-path、CSS Grid/Flexbox 布局、CSS 动画（演示文稿翻页）
- SVG：图标系统、装饰图形、数据可视化基元、架构图节点和连线
- Mermaid：流程图、时序图、类图、状态图、ER 图
- CSS 自定义属性：Theme Pack 变量注入的统一通道
- JS：交互逻辑、动态效果、数据驱动渲染

## 后果（Consequences）

### 正面
- 零运行时模态成本，结果 100% 可复现
- 矢量输出天然支持多分辨率和缩放
- 与 HTML 主格式（ADR-001）完美配合
- 用户零额外模态配置成本，开箱即用

### 负面
- 无法生成照片级视觉内容，海报/宣传图的视觉冲击力受限（可通过可选模态增强弥补）
- 需要持续积累 CSS/SVG 模式库来提升视觉丰富度

### 风险
- 用户期望与实际视觉表现力之间的落差
- 缓解：产品定位明确为"结构化设计产物"而非"图片生成"，通过版式设计和信息层级弥补视觉密度；可选模态增强为有更高视觉需求的用户提供升级路径
